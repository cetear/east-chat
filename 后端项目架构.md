# EasyChat 后端项目架构（实际实现）

## 一、整体 Maven 多模块结构

```
easychat-backend/
├── pom.xml                           # 父工程（依赖管理）
│
├── easychat-common/                  # 通用模块（工具类/常量/异常）
├── easychat-infra/                   # 基础设施（MySQL/ES/Kafka）
│
├── easychat-memory/                  # 记忆系统（会话上下文管理）
├── easychat-rag/                     # RAG 检索层（接口 + ES 实现）
├── easychat-tools/                   # 工具系统（Tool 注册与执行）
├── easychat-llm/                     # LLM 封装层（LangChain4j 客户端）
├── easychat-multimodal/              # 多模态处理（预留接口）
│
├── easychat-core/                    # Agent 核心（ReAct 循环 + 门面）
├── easychat-api/                     # 接口层（Controller/DTO）
├── easychat-app/                     # 启动模块（Spring Boot 入口）
└── easychat-test/                    # 集成测试
```

## 三、模块职责与实现

### 1. easychat-common（通用模块）

**依赖**：spring-boot-starter-web、Lombok、SLF4J

**实现文件**：

| 文件 | 职责 |
|------|------|
| `SSEUtil.java` | SSE 事件发送工具，支持 6 种事件类型：message/thought/action/observation/error/finish |
| `AIException.java` | 自定义 AI 异常类 |
| `MessageRole.java` | 枚举：USER、ASSISTANT、SYSTEM |
| `ModelType.java` | 枚举：OPENAI、CLAUDE、GEMINI、DEEPSEEK |
| `Result.java` | 通用响应包装类（code/message/data） |

**SSE 协议**：

```
event: message          # 逐 token 文本流（最终回答）
data: <token>

event: thought          # Agent 思考过程
data: {"content":"..."}

event: action           # Agent 工具调用
data: {"tool":"mysql_query","input":{...}}

event: observation      # 工具执行结果
data: {"content":"..."}

event: error            # 错误
data: <error_message>

event: finish           # 完成
data: stop
```

### 8. easychat-core（Agent 核心）

**依赖**：common、llm、tools、rag、memory

**实现文件**：

| 文件 | 职责 |
|------|------|
| `Agent.java` | 接口：`AgentResult run(AgentContext)` |
| `StreamingAgent.java` | 接口：继承 Agent，增加 `Flux<AgentEvent> streamRun(AgentContext)` |
| `AgentEvent.java` | 事件类：Type 枚举 + 静态工厂方法（thought/action/observation/message/error/finish） |
| `AgentResult.java` | 执行结果：response/success/errorMessage |
| `AgentContext.java` | 执行上下文：sessionId/userMessage/toolsEnabled/ragEnabled/maxIterations |
| `ReActAgent.java` | **核心实现**：ReAct 推理循环 |
| `ReActStep.java` | ReAct 单步解析结果 |
| `ReActOutputParser.java` | 正则解析 LLM 输出中的 Thought/Action/ActionInput/FinalAnswer |
| `AgentFacade.java` | 门面服务：协调 Agent、Memory、数据库、SSE、Kafka |

**ReAct 循环逻辑**：

```
当 toolsEnabled=false → 退化为简单 LLM 流式调用（向后兼容）
当 toolsEnabled=true：
  for (i = 0; i < maxIterations; i++) {
    1. 同步调用 llmClient.chat(prompt)
    2. ReActOutputParser 解析输出
    3. 发射 AgentEvent.thought()
    4. 如果有 FinalAnswer → 分块发射 AgentEvent.message() 并完成
    5. 如果有 Action → 发射 AgentEvent.action()
       → ToolRegistry.getTool().execute()
       → 发射 AgentEvent.observation()
       → 追加 Observation 到对话历史，继续循环
  }
```

**ReAct 系统提示词**：`prompts/react-system.txt`，指导 LLM 使用 Thought/Action/Action Input/Final Answer 格式。

## 四、模块依赖关系

```
easychat-app
    ↓
easychat-api
    ↓
easychat-core
    ↓
├── easychat-llm      ← LangChain4j + Reactor
├── easychat-tools     ← Tool 接口 + 注册
├── easychat-rag       ← 检索接口
├── easychat-memory    ← 会话记忆
├── easychat-multimodal ← 预留接口
    ↓
easychat-infra        ← MyBatis-Plus + MySQL + ES + Kafka
    ↓
easychat-common       ← SSEUtil + 异常 + 常量
```

**核心原则**：
- 单向依赖，禁止循环
- core 依赖抽象接口，不依赖具体基础设施
- infra 只提供数据访问能力，不参与业务逻辑
- ES 和 Kafka 通过 `@ConditionalOnProperty` 条件启用

## 五、调用链

```
ChatController (easychat-api)
    ↓
AgentFacade (easychat-core)
    ↓  保存用户消息 → 构建 AgentContext
ReActAgent (easychat-core)
    ↓  toolsEnabled=false → 简单 LLM 流式
    ↓  toolsEnabled=true  → ReAct 循环
LLMClient (easychat-llm)
    ↓  LangChain4j → OpenAI API
ToolRegistry → Tool.execute() (easychat-tools)
    ↓
ConversationMemory (easychat-memory)
    ↓
ChatMessageMapper / ChatSessionMapper (easychat-infra)
    ↓
MySQL 数据库
```

SSE 事件流方向：`ReActAgent` → `Flux<AgentEvent>` → `AgentFacade` → `SSEUtil` → 前端

## 六、技术要点

### 1. LangChain4j 替代 Python Agent

使用 `LangChain4jLLMClient` 封装 LangChain4j 的 `OpenAiChatModel` 和 `OpenAiStreamingChatModel`，在 Java 内部完成 LLM 调用，无需外部 Python 服务。

### 2. 结构化 SSE 流

`AgentEvent` 类型系统确保每个 SSE 事件都有明确语义。前端根据 `event:` 名称分发到不同渲染组件。

### 3. ReAct 循环的同步/流式混合

ReAct 推理阶段使用同步 `llmClient.chat()` 确保完整解析；最终回答阶段分块发送，模拟流式效果。

### 4. 工具系统的自动注册

实现 `Tool` 接口 + `@Component` 注解即可自动注册。`ToolRegistry` 在构造时通过 Spring 注入所有 Tool Bean。

### 5. 条件化中间件

ES 和 Kafka 使用 `@ConditionalOnProperty`，不配置连接地址时不创建 Bean，开发环境零依赖启动。
