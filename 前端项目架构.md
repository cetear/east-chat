## 三、核心数据结构

### 消息类型

```typescript
type MessageType = 'text' | 'thought' | 'action' | 'observation'

interface Message {
  id: string
  role: 'user' | 'assistant'
  type: MessageType
  content: string
  toolName?: string                 // type=action 时
  toolInput?: Record<string, unknown>  // type=action 时
  toolOutput?: string               // type=action 时
  timestamp: number
}
```

### 会话结构

```typescript
interface Session {
  id: number
  sessionId: string                 // UUID
  title: string
  modelType: string
  maxRounds: number
  messages: Message[]
  createdAt: number
  updatedAt: number
}
```

### SSE 事件

```typescript
interface AgentEvent {
  type: 'thought' | 'action' | 'observation' | 'message' | 'error' | 'finish'
  content?: string
  tool?: string
  input?: Record<string, unknown>
}
```

### 聊天请求

```typescript
interface ChatRequest {
  messages: Array<{ role: string; content: string }>
  model: string
  sessionId?: string
  toolsEnabled?: boolean
  ragEnabled?: boolean
}
```

## 四、SSE 流式协议

前端使用 `fetch` + `ReadableStream` 手动解析 SSE（非 EventSource，因为需要 POST 请求体）。

### 支持的事件类型

| 事件 | data 格式 | 处理方式 |
|------|-----------|---------|
| `message` | 纯文本 token | 追加到流式内容 |
| `thought` | JSON `{"content":"..."}` | 渲染 ThoughtBlock |
| `action` | JSON `{"tool":"...","input":{...}}` | 渲染 ToolCard |
| `observation` | JSON `{"content":"..."}` | 渲染 ObservationBlock |
| `error` | 纯文本错误信息 | ElMessage.error 提示 |
| `finish` | `stop` | 完成流式，保存消息 |

### SSE 回调接口

```typescript
interface SSECallbacks {
  onMessage?: (token: string) => void
  onThought?: (data: { content: string }) => void
  onAction?: (data: { tool: string; input: Record<string, unknown> }) => void
  onObservation?: (data: { content: string }) => void
  onError?: (msg: string) => void
  onFinish?: () => void
}
```

## 五、状态管理（Pinia）

### chatStore

| 状态 | 类型 | 说明 |
|------|------|------|
| `sessions` | `Session[]` | 所有会话列表 |
| `activeSessionId` | `string` | 当前激活会话 UUID |
| `currentMessages` | `Message[]` | 当前会话消息列表 |
| `isStreaming` | `boolean` | 是否正在流式响应 |
| `streamContent` | `string` | 流式累积的文本内容 |
| `streamingEvents` | `Message[]` | 流式过程中的 Agent 事件 |

| 方法 | 说明 |
|------|------|
| `loadSessions()` | 加载会话列表，无则自动创建 |
| `createSession(modelType)` | 创建新会话 |
| `switchSession(sessionId)` | 切换会话并加载消息 |
| `deleteSession(sessionId)` | 删除会话 |
| `addUserMessage(content)` | 添加用户消息 |
| `addStreamingEvent(event)` | 添加 Agent 事件（思考/工具/观察） |
| `appendStreamContent(token)` | 追加流式文本 |
| `finalizeAssistantMessage()` | 流式结束，合并为完整 AI 消息 |

### configStore

| 状态 | 默认值 | 说明 |
|------|--------|------|
| `currentModel` | `'openai'` | 当前模型 |
| `isStreamingMode` | `true` | 流式/非流式输出 |
| `toolsEnabled` | `false` | 是否启用 Agent 工具 |
| `ragEnabled` | `false` | 是否启用 RAG 检索 |
| `modelConfigVisible` | `false` | 配置弹窗可见性 |

## 六、API 接口

### 聊天与会话

| 函数 | 方法 | 端点 | 说明 |
|------|------|------|------|
| `streamChat(data)` | POST | `/api/chat/stream` | 流式聊天，返回 Response |
| `chat(data)` | POST | `/api/chat` | 同步聊天 |
| `createSession(modelType)` | POST | `/api/session` | 创建会话 |
| `getSessions()` | GET | `/api/sessions` | 获取所有会话 |
| `getSession(id)` | GET | `/api/session/:id` | 获取单个会话 |
| `updateSession(id, data)` | PUT | `/api/session/:id` | 更新会话 |
| `deleteSession(id)` | DELETE | `/api/session/:id` | 删除会话 |

### 模型配置

| 函数 | 方法 | 端点 | 说明 |
|------|------|------|------|
| `getModels()` | GET | `/api/config/models` | 获取模型列表 |
| `getCurrentModel()` | GET | `/api/config/current` | 获取当前模型 |
| `setCurrentModel(model)` | POST | `/api/config/current` | 设置当前模型 |
| `getModelConfig(code)` | GET | `/api/config/model/:code` | 获取模型配置 |
| `saveModelConfig(config)` | POST | `/api/config/model` | 保存模型配置 |
| `updateModelConfig(code, config)` | PUT | `/api/config/model/:code` | 更新模型配置 |

## 七、组件详解

### 核心组件

#### ChatView.vue（主页面）

布局：左侧 ChatSidebar（280px）+ 右侧主区域（header + 消息列表 + 输入框）

Header 控件：
- 流式/非流式切换按钮组
- Tools 开关（El-Switch）
- 模型选择器
- 配置按钮

消息发送流程：
1. 用户输入 → `chatStore.addUserMessage()`
2. 自动更新会话标题（取前 20 字）
3. 流式模式 → `streamChat()` + `handleStreamChat()` 回调分发
4. 非流式模式 → `chat()` 同步获取

#### ChatMessage.vue（消息渲染）

按 `message.type` 分发：
- `thought` → ThoughtBlock
- `action` → ToolCard
- `observation` → ObservationBlock
- 默认 → Markdown 渲染（`marked.parse()`）

用户消息右对齐蓝色气泡，AI 消息左对齐灰色气泡。

#### StreamMessage.vue（流式响应容器）

流式过程中显示，包含三部分：
1. Agent 事件序列（ThoughtBlock / ToolCard / ObservationBlock）
2. 最终回答的 Markdown 流式渲染
3. 加载指示器

#### ChatSidebar.vue（会话侧边栏）

功能：
- 分页会话列表（每页 10 条）
- 右键菜单（重命名/删除）
- 重命名弹窗
- 相对时间显示（"刚刚"/"X 分钟前"等）
- 模型类型标签（OpenAI/Claude/Gemini/DeepSeek）
- 新建会话按钮

### Agent 可视化组件（新增）

#### ThoughtBlock.vue

黄色/琥珀主题，可折叠，脑图标 + "Thinking..." 标签。显示 Agent 的推理过程。

#### ToolCard.vue

蓝色主题，齿轮图标。显示工具名称、JSON 格式的输入参数、执行结果。

#### ObservationBlock.vue

绿色主题，剪贴板图标。显示工具执行返回的观察结果。

#### RagReference.vue（预留）

紫色主题，可折叠。显示 RAG 检索引用的文档列表（标题/内容/相关度分数）。**已实现组件但未接入页面**。

### 配置组件

#### ModelSelector.vue

Element Plus 下拉框，从 `/api/config/models` 获取模型列表。

#### ModelConfig.vue

Element Plus 弹窗表单，配置模型参数（API Key、Base URL、Temperature、Max Tokens 等）。

#### ChatInput.vue

多行文本框 + 发送按钮 + 文件上传按钮（disabled 预留）。Enter 发送，Shift+Enter 换行。
